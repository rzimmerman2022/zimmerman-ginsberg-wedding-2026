## 3. Technical Decisions Made

### Decision 1: Complete Directory Flattening
**Context:** Repository had organized subdirectory structure (docs/, research/, guests/, budget/, etc.)

**Decision:** Flatten ALL wedding planning files to root directory, preserving only infrastructure folders

**Rationale:**
- User explicitly requested flat structure for simplicity
- Non-technical users (Ryan & Jordyn) benefit from single-location file access
- Reduces cognitive load for daily wedding planning tasks
- Mobile/cloud accessibility improved

**Trade-offs Accepted:**
- Reduced organization and scalability
- Root directory will have 15+ files
- May need reorganization in 3-6 months as project grows
- Goes against software engineering best practices

**Alternatives Considered:**
1. Keep original subdirectory structure (better organization, rejected)
2. Hybrid 2-3 folder approach (still requires navigation, rejected)
3. Use naming prefixes for grouping (could revisit later, deferred)

**Documented in:** [ADR_2025-10-25_001_flatten-directory-structure.md](docs/decisions/ADR_2025-10-25_001_flatten-directory-structure.md)
